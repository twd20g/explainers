Core concepts
Prior P(C)
How common is class C in training data?
Likelihood P(xᵢ | C)
How often does feature xᵢ appear in class C?
Posterior P(C | x)
Given features x, probability of class C?
Decision rule
Pick the class with the highest posterior

Training likelihoods (Laplace smoothed): free: spam 0.80, ham 0.10 · win: spam 0.70, ham 0.05 · lunch: spam 0.05, ham 0.60 · meeting: spam 0.05, ham 0.70 · Prior: P(spam)=0.4, P(ham)=0.6

Email contains words: free win. Classify as spam or ham?

Step 1 — Prior probabilities
P(spam) = 40/100 = 0.40    P(ham) = 60/100 = 0.60
Step 2 — Likelihoods (Laplace smoothed)
P(free | spam) = 0.80   P(free | ham) = 0.10
P(win  | spam) = 0.70   P(win  | ham) = 0.05
Step 3 — Unnormalised posteriors (log-space)
score(spam) = log(0.40) + log(0.80) + log(0.70)
             = −0.916 + (−0.223) + (−0.357) = −1.496

score(ham)  = log(0.60) + log(0.10) + log(0.05)
             = −0.511 + (−2.303) + (−2.996) = −5.810
Step 4 — Normalise to probabilities
exp(−1.496) = 0.224    exp(−5.810) = 0.003
sum = 0.227

P(spam | email) = 0.224 / 0.227 = 98.7%
P(ham  | email) = 0.003 / 0.227 =  1.3%
Prediction: Spam — with 98.7% confidence

Log-space arithmetic prevents numerical underflow when multiplying many small probabilities — essential in practice.

!
Conditional independence
Features are assumed independent given the class. Despite frequent violations in real data, NB often works well in practice — the "naïve Bayes paradox". Decision boundaries only require correct ordering of posteriors, not exact values.
!
Probability calibration
Raw posterior probabilities are often poorly calibrated due to the independence assumption. Useful for ranking and thresholding, not always for absolute probability estimates.
✓
Laplace (add-1) smoothing
Prevents zero-probability problem for unseen features. Replaces P(x|C) = count/N with (count + α)/(N + α·|V|). Essential when vocabulary is large relative to training data.
✓
Variants for different feature types
Bernoulli NB — binary feature presence. Multinomial NB — word counts. Gaussian NB — continuous features (Gaussian likelihood). Mix these for heterogeneous feature sets.
